**11/9**

우리 포춘디어 멤버들은 항상 목요일에 회의를 한다.

내가 회의 목차와 일정을 알려주면 다들 준비해온다.

그리고 이번 목차는


```
1.기능명세서&API명세서 수정

2.대표캐릭터 아이디어 수정

3.프론트와 백엔드 문제 논의
```

였다

1번과 3번은 잘 되었다.

근데 항상 문제는 대표 캐릭터이다.

대중성 따지고 세세한 컨셉을 추가 수정을 반복한다.

그래서 이번에 또 바뀌었다

```
포춘쿠키 모양 머리핀 -> 포춘쿠키 모양 귀걸이
```

항상 포춘쿠키를 캐릭터와 연관시키는 것이 힘들었다.

모자, 핸드백, 귀걸이, 머리핀, 옷 등

다양한 선택지 중 다른 사람들(친구들)을 투표에 참가시켜

결국 귀걸이로 바꾸었다.

```
-정보-
```

요즘 학교에서 내는 과제가 너무 많다 - ex) java, 자료구조 프로젝트, ppt 정리 및 발표 등

요즘 연말이라 그런지 모두 바뻐 포춘디어의 진행 상황이 점점 느려져 가고 있다.

그리고 우리는 내년 즉 2024년 6월달 까지 포춘디어를 완성시키기로 하였다.

그리고 나머지 6개월은 유지 및 보수를 할 것이고

지금은 html을 배우지만 1월달 부터 React를 사용해 만들 예정이다.

회의(나의 통보?)를 통해 결정된 사안이다.
______________________________________________________________________________
**컨셉**

**이름** : 웨어

**성별** : 여자

**머리색** : 연한 노란색 [파스텔톤]

**눈 색** : 연한 하늘색 or 연한 연두색 [파스텔톤]

**귀걸이** : 포춘쿠키 모양 귀걸이[포춘디어의 포춘쿠키를 연상시키기 위함]

**뱃지** : 네잎클로버 [포춘쿠키 영어 → fortunecookie의 fortune(운)을 상징하는 용도]

**2024/1/4**

**포춘디어를 갈아엎을 것이다.**

포춘디어 프로젝트 자체가 아닌 디자인, 사용 언어 등 말이다

이유는

> 1. 디자인이 구림(급조함)
> 2. 화면 비율이 figma와 클라이언트가 다름
> 3. html보다 React가 유지/보수에 좋음
> 4. 명세서 작성이 제대로 되어 있지 않았음

정도가 있다.

1번 문제는 웹의 **대표 색**도 다시 정하고 디자인 자체도 많이 바뀔 예정이다.

2번 문제는 figma의 frame desktop이라는 보기가 있는데 가로 1440 세로 1024인데 실제 클라이언트와 같은 비율은 가로 1920 세로 1080이라고 한다.

3번 문제는 그냥 html/css+ js보다는 React가 훨씬 유지/보수도 편하고, 언제까지고 html만 쓸 순 없는데다 퀄리티 문제도 있어 바꾸려고 한다.

4번 문제는 가장 중요한데 명세서 작성이 제대로 되어있지 않았다. 예를 들어 API명세서(백엔드 관련 명세서)를 내가 적었다던가?(나 : 프론트) 등 많은 문제가 있었다.

이러한 이유로 확 갈아엎을 것이다.

그런데 백엔드 팀원 중 **누군가**가 저번주에 80% 정도 완성된 기능 명세서를 지워버렸다.

우리 포춘디어 팀은 공동 노션을 사용하는데 노션을 평소에 사용하지 않은 **누군가**가

표의 +를 눌렀는데 변화가 없자. -를 통해 지워버린 것이다.

(표는 +를 누르면 새로운 표가 추가되는 게 아닌 표의 공간이 +되는 것을 몰랐던 거임 **누군가**가)

(익명을 위해 **누군가**라고 표현하고 있지만 누군가의 정체는 다들 알고 있는 상황)

## GDWEB이란?

[**GDWEB**](https://www.gdweb.co.kr/main/index.asp)이라는 UXUI쪽 으로 좋은 정보 또는 게시물을 보고 참고할 수 있는 사이트이다.

난 이 사이트에서 내가 개인적으로 생각하기에 UI부분에서 꽤 좋다고 평가하는

[**치지직**](https://chzzk.naver.com)이라는 사이트를 참고하였다.

## 대표캐릭터에 대해

대표 캐릭터는 없애기로 하였다.

웹사이트에 필수 요소가 아닌 것도 있지만

오히려 디자인에 방해된다는 생각도 있어서 이다.

당연하게도 단순하고 보기 쉬운 디자인은 사람들에게 먹힌다.

하지만 굳이 거기에 비주류이고 단순함을 망치는 캐릭터를 넣을 필요가 없다고 생각하기 때문인데

캐릭터를 추가하려던 이유는 상징성을 만들기 위함인데

아이디어 나눔이라는 목적과 대표 색상 정도만으로도 이미 상징성을 갖춘 것 같다는 생각도 든다.
